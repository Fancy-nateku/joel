Semantic HTML

Why Semantics Matter

Semantic elements clearly describe their meaning to both the browser and the developer. Examples include <article>, <section>, and <aside>. They improve accessibility, SEO, and maintainability.

Common Semantic Elements

  • <header>: Introductory content or navigational links.
  • <main>: Main content of the document.
  • <footer>: Footer for the document or a section.
  • <nav>: A section of navigation links.
  • <aside>: Content related to the main content (like side notes or tips).

Tip: Avoid redundant ARIA roles on native landmark elements. For example, <main> already exposes the main landmark to assistive technologies.

Accessibility

Using ARIA and Landmarks

Prefer semantic HTML first, then add ARIA only when necessary. Landmarks like <header>, <nav>, <main>, and <footer> help screen reader users navigate quickly.
